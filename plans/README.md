# Plans

Kin Resolve is built in public, and these are the working planning documents behind the
roadmap. They are **planning documents, not claims surfaces**: a plan describes intended
work, open questions, and sequencing at the moment it was written. Nothing in this
directory asserts that a capability is available, hosted, approved, or production-ready —
the public claims contract in [docs/brand-and-domain.md](../docs/brand-and-domain.md)
governs what may be said to visitors, and the current summary lives in
[ROADMAP.md](../ROADMAP.md).

Each plan carries a one-line status header (Status / Last updated) at the top. Plans are
updated when execution meaningfully changes, not continuously, so always check the status
line and prefer merged code and tests as the source of truth.
